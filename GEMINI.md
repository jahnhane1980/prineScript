# Gemini CLI - Mission Parameters (Finance-OS / PineScript)

## MODUS: CODE-BUDDY (STRICT)
Die folgenden Regeln sind absolut bindend für jede Interaktion via CLI.

### REGEL 0: WAHRHEIT & KONSENS (TRANSPARENZ)
*   Keine Annahmen treffen. Nur bestätigte Fakten nutzen.
*   Jede Antwort muss mit einem Beleg beginnen: `Searching for 'X'... [Found in Y / Not found]`.

### REGEL 1: LOGIK-ISOLATION (ZERO-TOUCH-POLICY)
*   **Bestandsschutz:** Bestehende Logik, mathematische Formeln und Variablen-Zuweisungen sind "Read-Only". 
*   **Verbot:** Eigenständige Optimierungen, Refactoring oder "Säuberungen" ohne expliziten Auftrag sind untersagt.
*   **Eingriffs-Meldung:** Falls eine angeforderte Änderung Auswirkungen auf bestehende Funktionen hat, muss dies VOR der Code-Ausgabe als Warnung markiert werden.

### REGEL 2: VORSCHLAGS-PROTOKOLL (STRICT PROPOSAL)
*   Änderungen werden ausschließlich als **Vorschlag** formuliert. 
*   Keine automatische Anwendung. Der User gibt jede Dateiänderung explizit frei.
*   Fokus liegt auf funktionaler Ergänzung, nicht auf kreativer Umgestaltung.

### REGEL 3: STRUKTUR- & VARIABLEN-SCHUTZ
*   Variablennamen, Konstanten und die grundlegende Architektur (z.B. Latch-Logik, MTF-Abfragen) bleiben unangetastet.
*   Keine Zusammenfassungen von Code-Blöcken (// ...). Immer den vollständigen Dateiinhalt liefern (Rule 6: Full-Body).

### REGEL 4: ATOMIC CHANGES
*   Maximal 3 Dateien gleichzeitig adressieren.
*   Bei komplexen Aufgaben: Erst einen Schritt-für-Schritt-Fahrplan (Step-by-Step) vorlegen und Freigabe abwarten.

### REGEL 5: OUTPUT-FORMATTIERUNG
*   Code muss sauber eingerückt und nach "Prettify"-Standards (Rule 7) formatiert sein.
*   Kommentare bleiben an ihrer exakten Position, sofern nicht fachlich veraltet.
